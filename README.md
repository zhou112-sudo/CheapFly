# TicketPlatform 产品与操作说明

## 1. 项目简介

TicketPlatform 是一个机票搜索与价格洞察 Demo，核心目标是帮助用户快速完成「出发地 -> 到达地 -> 日期」搜索，并在结果页查看推荐方案、价格参考与完整行程列表。

当前项目由前后端两部分组成：

- `frontend`：React + Vite + Tailwind + Radix/shadcn 风格组件
- `backend`：FastAPI，提供 `/search` 接口（基于 mock 数据）

后端机场基础数据使用 `airportsdata`（IATA）作为数据源；当静态 mock 航班不足时，系统会动态生成示例航班，提升可查询覆盖率。

前端「出发地 / 到达地」选择器使用**独立的本地机场目录**（`frontend/src/data/airportCatalog.json`），用于搜索与展示；与后端 mock/动态航班数据源解耦，便于扩充与演示。

## 2. 核心功能总览

### 2.1 搜索页能力

- 单程 / 往返模式切换（默认单程）
- 出发地选择（支持城市“所有机场”与单机场）
- 到达地选择（支持城市“所有机场”与单机场）
- 出发/到达搜索：基于本地 `airportCatalog.json`，支持中国**省份**名、国家名（中文及常见英文别名）、城市中文/英文名、机场名称、IATA 三字码及 `keywords`；省份与国家不单独作为可选项，仅用于筛出下属「城市 → 机场」层级；多关键词空格分隔时为“全部子串命中”；无匹配时展示纯文案「无结果」
- 去程日期选择（自定义双月日历面板）
- 返程日期选择（往返模式下显示）
- 机票种类选择（经济舱 / 超级经济舱 / 公务舱 / 头等舱）
- 出发/到达中间交换按钮
- 搜索按钮状态控制：
  - 未填写出发或到达时按钮置灰且不可点击
  - 填写完整后可点击搜索

### 2.2 搜索输入约束

- 出发与到达不允许为同一城市
- 当用户尝试选择同城时，系统阻止更新并提示错误信息
- 去程、返程（往返）日期不得早于**本地**当天；早于今天时提示「不能查询过去日期的航班。」且不进入结果页

### 2.3 搜索条件保留（UX 优化）

- 搜索页输入状态通过 URL query 参数持久化：
  - `originId`
  - `destinationId`
  - `date`
  - `tripType`
  - `returnDate`
- `cabinClass`
- 用户进入结果页后返回搜索页，先前选择会直接恢复，无闪烁、无先空后填

### 2.4 结果页能力

- 列表仅展示「出发时刻 ≥ 本地当前时间 + 1 小时」的航班（当日查询会过滤已过期或过近班次）；若过滤后无行程，正常展示空态说明（如「当前时间范围内暂无可预订航班」），不作为接口错误
- 顶部查询摘要：
  - 出发、到达、日期、总行程数量
- 推荐方案卡片
- 价格参考卡片（当前最低、近期均价、历史低价示意；**单张**「最低价走势与预测」折线图：左侧 Y 轴价格刻度、黑白灰主色、历史实线 / 预测虚线、轻量竖线分隔；横轴为自然相对时间（「今天」仅在横轴标签、小字弱化）；不再展示历史最高价；状态标签判定基于「当前最低 vs 历史低价与近期均价」；绿 / 橙 / 红浅色标签）
- 最低价摘要（全场最低 / 直飞最低 / 中转最低）
- 实惠中转推荐
- 全部行程列表（可展开、可预订）

## 3. 主要页面说明

### 3.1 搜索页（`/`）

用于填写搜索条件并发起查询。输入区域已做以下体验优化：

- 结构清晰：单程/往返切换 + 出发、到达、去程/返程日期分组展示
- 标签外置：出发/到达/日期标签位于输入框外，桌面端地点列更宽、日期列更窄
- 机场展示：选择具体机场时，触发器展示机场全称（与列表一致）
- 交互连贯：日期选择器与整体样式一致，不使用原生浏览器日期控件样式
- 状态可靠：用户返回页面后，输入条件自动恢复

### 3.2 结果页（`/results?...`）

用于展示搜索结果与价格信息，支持用户继续返回首页修改搜索条件后再次查询。

- 顶部「出发 → 到达」标题与搜索页选择一致：若用户选的是城市节点（`kind: "city"`），展示为「城市名（所有机场）」；若为单机场（`kind: "airport"`），展示为机场全称与三字码；**不会**因某城市仅有一个机场或航班落在某一机场而把城市选择改成单机场文案。（直链打开结果页、仅有 `origin_airports` 参数而无路由 state 时，仍按机场代码做摘要推断。）

## 4. 参数与状态结构

### 4.0 前端机场目录（`airportCatalog.json`）

用于搜索页地点选择器，结构如下：

- 根对象：
  - `version`：整数，目录格式版本
  - `description`：可选说明
  - `cities`：城市数组
- 每个城市（`cities[]`）：
  - `id`：稳定 ID（与 URL 中 `city:${id}` 对应）
  - `name` / `nameEn`：城市中文名、城市英文名（参与搜索，对应需求中的 city/city_zh 与城市英文检索）
  - `country`：国家或地区展示名（中文，参与搜索，对应 country/country_zh）
  - `province`：可选；中国省份中文名（参与搜索，**不作为下拉可选项**，仅用于筛出省内城市）
  - `countryEn`：可选；非中国国家名的英文检索别名串（小写词组，参与搜索；与 `country` 并存）
  - `metroCode`：城市/都会三字码（展示与搜索）
  - `keywords`：可选补充搜索词（如俗称、三字码变体）
  - `airports`：该城市下属机场列表
- 每个机场（`airports[]`）：
  - `code`：IATA 三字码（大写存储）
  - `name` / `nameEn`：机场通称中文、英文名（参与搜索，对应 airport 名称维度）
  - `pickerName`：可选；若提供，则列表、触发器与结果摘要中该机场以该全称为准（避免非「国际机场」命名硬套后缀）

维护方式：

- 直接编辑 `frontend/src/data/airportCatalog.json`，或
- 修改 `frontend/scripts/build-airport-catalog.mjs` 后执行：`npm run gen:airports`（在 `frontend` 目录下）

## 4.1 搜索页 URL 参数（用于状态保留）

- `originId`: 出发地选择项 ID（如 `city:beijing`、`ap:PEK`）
- `destinationId`: 到达地选择项 ID（如 `city:shanghai`、`ap:PVG`）
- `date`: 去程日期（`YYYY-MM-DD`）
- `tripType`: 行程类型（`oneway` / `roundtrip`）
- `returnDate`: 返程日期（`YYYY-MM-DD`，仅往返）
- `cabinClass`: 机票种类（`economy` / `premium_economy` / `business` / `first`）

示例：

`/?originId=city:beijing&destinationId=city:shanghai&date=2026-04-14&tripType=roundtrip&returnDate=2026-04-18&cabinClass=business`

### 4.2 结果页接口查询参数（用于后端搜索）

- `origin_airports`
- `destination_airports`
- `departure_date`
- `cabin_class`

示例：

`/results?origin_airports=PEK,PKX&destination_airports=PVG,SHA&departure_date=2026-04-14&cabin_class=economy`

## 5. 用户操作指南

### 5.1 基本搜索流程

1. 打开首页
2. 点击“出发”，选择城市或机场
3. 点击“到达”，选择城市或机场
4. 点击“出发日期”，在日历中选择日期
5. 点击“搜索”进入结果页

### 5.2 往返搜索流程

1. 在搜索区切换到“往返”
2. 选择出发与到达
3. 选择去程日期和返程日期
4. 系统校验：去程日期必须早于返程日期
5. 点击“搜索”进入结果页

### 5.3 修改后重搜

1. 在结果页点击返回（或浏览器返回）
2. 搜索页会保留上次输入
3. 调整任一条件后再次搜索

### 5.4 使用交换按钮

1. 在出发与到达中间点击交换按钮
2. 系统将互换出发和到达内容
3. 若交换后触发同城限制，按页面提示重新选择

### 5.5 常见错误提示

- 未填写出发/到达：按钮不可点击（灰色）
- 出发与到达同城：提示“出发和到达不能为同一城市，请重新选择。”
- 往返未填写返程日期：提示“往返查询请填写返程日期。”
- 去程晚于或等于返程：提示“去程日期必须早于返程日期。”

## 6. 本地运行与预览

### 6.1 启动后端

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

### 6.2 启动前端

```bash
cd frontend
npm install
npm run dev
```

默认通过 Vite 代理访问后端：

- 前端请求 `/api/search`
- 代理到 `http://127.0.0.1:8000/search`

### 6.3 生产构建

```bash
cd frontend
npm run build
npm run preview
```

## 7. 设计与交互原则（当前版本）

- 风格：简洁、黑白灰、圆角、轻阴影
- 层级：主信息高对比，次要信息使用灰色
- 可读性：搜索输入与结果信息分层明确
- 一致性：弹层、卡片、表单遵循统一样式语言
